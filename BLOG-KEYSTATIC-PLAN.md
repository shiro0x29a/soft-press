# План реализации блога на Keystatic

## Что такое Keystatic
- Git-based CMS, контент хранится как Markdown/JSON файлы в репозитории
- Нет базы данных — всё на файлах
- Встроенная админка с rich text editor (на базе TipTap/ProseMirror)
- Работает локально и через интеграцию с GitHub

## Этапы реализации

### 1. Установка
```bash
npm install @keystatic/core @keystatic/next @keystatic/ui
```

### 2. Конфигурация (`keystatic.config.ts`)
- Определить коллекцию `posts`
- Схема:
  - `title` (текст)
  - `slug` (слаг)
  - `content` (markdown — rich text редактор)
  - `excerpt` (текст, опционально)
  - `coverImage` (изображение, опционально)
  - `tags` (множественный выбор)
  - `author` (текст/ссылка)
  - `publishedAt` (дата)
  - `status` (enum: черновик/опубликовано)
- Хранилище: локальное (файлы в `/content/posts/`)

### 3. Роуты админки
- `/admin/[[...params]]` — Keystatic Admin UI
- Встроенный редактор, превью, управление файлами
- Не требует аутентификации для локальной разработки (добавить auth для продакшена)

### 4. Чтение постов в приложении
- Использовать `@keystatic/reader` для серверных компонентов
- Парсинг Markdown → HTML/React
- Статическая генерация (`generateStaticParams`)

### 5. Страницы блога
- `/blog` — список всех постов (с фильтрацией, пагинацией)
- `/blog/[slug]` — отдельный пост
- Фильтрация по тегам, автору, статусу
- RSS лента (опционально)

### 6. Деплой
- **Локальный режим**: файлы коммитятся вручную
- **GitHub режим**: Keystatic сам создаёт коммиты через API
- Деплой на Vercel / любой хостинг

## Плюсы Keystatic
- ✅ Готовая админка из коробки
- ✅ Не нужна база данных
- ✅ Rich text editor встроен (на базе TipTap)
- ✅ Контроль версий через Git
- ✅ Работает офлайн
- ✅ Типизированные схемы (в стиле Zod)

## Минусы Keystatic
- ❌ Контент = файлы в репо (не для больших команд)
- ❌ Нет ролей/прав доступа (без интеграции с GitHub)
- ❌ Нет комментариев/форума из коробки
- ❌ Медленнее при очень большом количестве постов

## Структура файлов после реализации
```
/content/posts/
  my-first-post/
    index.md
    index.json (frontmatter)
/keystatic.config.ts
/src/app/admin/[[...params]]/page.tsx
/src/app/blog/page.tsx
/src/app/blog/[slug]/page.tsx
/src/lib/blog.ts
```
