# План реализации блога с TipTap редактором

## Стек технологий

| Слой | Технология |
|---|---|
| **Фреймворк** | Next.js 16 (App Router) |
| **Редактор** | TipTap (headless rich text editor) |
| **База данных** | PostgreSQL + Drizzle ORM |
| **Хранилище изображений** | Vercel Blob / Cloudflare R2 / AWS S3 |
| **Аутентификация** | Существующая NextAuth.js (admin роль) |
| **Стили** | Tailwind CSS 4 + shadcn/ui |
| **Валидация** | Zod |

---

## Фаза 1: База данных и инфраструктура

### 1.1 Установка зависимостей
```bash
npm install drizzle-orm pg
npm install -D drizzle-kit @types/pg
```

### 1.2 Настройка Drizzle ORM
- Создать `drizzle.config.ts`
- Добавить `DB_URL` в `.env.example`
- Создать `src/shared/lib/db.ts` (клиент Drizzle)

### 1.3 Схема базы данных
Файл `src/shared/db/schema.ts`:

```
Таблицы:
- posts: id, slug, title, content (JSON), excerpt, coverImage, 
         status (draft/published), authorId, createdAt, updatedAt, publishedAt
- categories: id, name, slug
- postCategories: postId, categoryId (many-to-many)
- tags: id, name, slug
- postTags: postId, tagId (many-to-many)
- images: id, url, alt, postId (nullable), uploadedBy, createdAt
```

### 1.4 Миграции
```bash
npx drizzle-kit generate
npx drizzle-kit migrate
```

## Фаза 2: Хранилище изображений

### 2.1 Провайдер
**Рекомендую: Vercel Blob** (проще всего для Vercel)
```bash
npm install @vercel/blob
```

Альтернатива: Cloudflare R2 или AWS S3

### 2.2 API для загрузки изображений
`src/app/api/upload/image/route.ts`:
- Принимает multipart/form-data
- Валидация типа и размера файла (Zod)
- Загрузка в хранилище
- Возвращает `{ url, alt }`

### 2.3 API для удаления изображений
- Принимает URL изображения
- Удаляет из хранилища
- Удаляет из БД если отслеживается

## Фаза 3: TipTap редактор

### 3.1 Установка зависимостей
```bash
npm install @tiptap/react @tiptap/starter-kit @tiptap/extension-image \
  @tiptap/extension-link @tiptap/extension-placeholder \
  @tiptap/extension-underline @tiptap/extension-text-align \
  @tiptap/extension-typography @tiptap/extension-table \
  @tiptap/extension-table-header @tiptap/extension-table-cell \
  @tiptap/extension-code-block-lowlight lowlight
```

### 3.2 Компонент редактора
`src/features/blog/components/tiptap-editor.tsx`:
- Панель инструментов (жирный, курсив, подчёркивание, заголовки, списки, код, цитаты, ссылки, изображения, таблицы, выравнивание)
- Кнопка загрузки изображений → вызывает API загрузки
- Drag & drop изображений
- Текст-заполнитель
- Поддержка режима редактирования/только для чтения

### 3.3 Обработчик загрузки изображений
`src/features/blog/lib/image-upload.ts`:
- Валидация файла
- Загрузка через API
- Вставка в редактор как `<img>` узел

### 3.4 Рендерер контента
`src/features/blog/components/tiptap-renderer.tsx`:
- Клиентский компонент для отображения TipTap JSON/HTML на публичных страницах
- Хранение контента в формате HTML или JSON

## Фаза 4: Админ-панель блога (защищённые роуты)

### 4.1 Структура роутов
```
src/app/(protected)/@admin/
├── dashboard/              # Существующий дашборд
├── blog/
│   ├── page.tsx            # Список постов (таблица с действиями)
│   ├── new/
│   │   └── page.tsx        # Создание нового поста (TipTap редактор)
│   └── [id]/
│       └── page.tsx        # Редактирование существующего поста
```

### 4.2 Страница списка постов
- Таблица: заголовок, статус (черновик/опубликован), категория, дата
- Действия: Редактировать, Удалить, Предпросмотр
- Фильтры по статусу, категории, тегам
- Поиск по заголовку
- Пагинация

### 4.3 Страница создания/редактирования
- Поля формы: title, slug (автогенерация), excerpt, обложка, контент (TipTap), категории, теги
- Автосохранение черновика (опционально)
- Кнопки: Сохранить как черновик / Опубликовать
- Кнопка предпросмотра (открывает `/blog/[slug]?preview=true`)
- Валидация через Zod

### 4.4 Server Actions
`src/features/blog/actions/posts.ts`:
- `createPost(data)` — валидация Zod, вставка в БД
- `updatePost(id, data)` — валидация, обновление
- `deletePost(id)` — мягкое или полное удаление
- `publishPost(id)` — установка статуса + publishedAt
- Все действия проверяют `session.user.role === 'admin'`

## Фаза 5: Публичные страницы блога

### 5.1 Структура роутов
```
src/app/(public)/
├── blog/
│   ├── page.tsx            # Список блога (пагинация, только опубликованные)
│   ├── feed.xml/
│   │   └── route.ts        # RSS лента
│   └── [slug]/
│       └── page.tsx        # Отдельный пост
└── sitemap.ts              # Обновить с включением постов блога
```

### 5.2 Страница списка блога
- Сетка карточек постов
- Карточка: обложка, заголовок, краткое описание, дата, категории, время чтения
- Пагинация (10-20 на страницу)
- Фильтр по категории/тегу
- Поиск
- SEO: meta title, description, OG теги

### 5.3 Страница отдельного поста
- Заголовок, автор, дата, категории, теги
- Обложка
- Контент из TipTap (HTML)
- Оглавление (автогенерация из заголовков)
- Примерное время чтения
- Кнопки соцсетей
- Похожие посты (те же категории/теги)
- Комментарии (Фаза 7 - опционально)
- SEO: JSON-LD Article схема

### 5.4 Получение данных
- Серверные компоненты с прямыми запросами к БД
- `getPublishedPosts()` — постраничный список
- `getPostBySlug(slug)` — отдельный пост
- Кэширование с `unstable_cache`

## Фаза 8: Тестирование

### 8.1 Юнит-тесты (Vitest)
- Компонент TipTap редактора (действия панели, загрузка изображений)
- Server Actions (валидация, авторизация)
- Утилиты (генерация slug, расчёт времени чтения)

### 8.2 E2E тесты (Playwright)
- Админ создаёт пост → пост появляется в публичном списке
- Админ редактирует пост → изменения отображаются
- Админ удаляет пост → удалён из списка
- Публичный пользователь просматривает пост → контент рендерится корректно
- Поток загрузки изображений

### 8.3 Интеграционные тесты
- API загрузки (валидация файлов, хранилище)
- Запросы к БД (пагинация, фильтрация)

## Ключевые решения, которые нужно принять

1. **База данных**: PostgreSQL (рекомендую) или SQLite/MySQL?
2. **Хранилище изображений**: Vercel Blob, Cloudflare R2 или AWS S3?
3. **Формат контента**: Хранить TipTap как JSON или HTML в БД?
   - JSON: Гибче, легче мигрировать
   - HTML: Быстрее рендерить, менее портативный
4. **Комментарии**: Сторонний сервис (Disqus) или собственный?
5. **Автосохранение**: Реал-тайм или только ручное сохранение?
6. **i18n**: Многоязычный блог или один язык?
